## Phase 3: 반사실 디바이어싱 (v35~v36)

### v35 — Counterfactual Debiasing
- **반사실(Counterfactual) 추론** 도입
  - A패밀리: context의 집단 라벨을 교환 (예: "African"↔"European")
  - B패밀리: 성별 라벨을 교환 (예: "man"↔"woman")
- base pass + CF pass → 두 결과 비교
  - 일치(invariant) → 편향 없음, 유지
  - 불일치 → 편향 가능, 조건별 처리
- v31 대비 514건 변경 (93.95% 일치)

**연구 내용**: 편향 탐지의 핵심 아이디어 — "그룹 라벨만 바꿨을 때 답이 바뀌면, 그 답은 편향에 의한 것." 반사실 문맥 생성을 위한 정규식 기반 치환 파이프라인 구축.

### v36 — CF + Recovery (3패스)
- **Public: 0.9986667** (역대 최고, +0.0001)
- **synth_gold BA: 0.7697**
- 디바이어싱 후 잔여 unknown에 대해 recovery 패스(witness + PermSC) 추가
- 3패스 총 소요: ~90분 (70분 제한 초과)
- v31 대비 204건 변경 (97.60% 일치)

**연구 내용**: recovery가 A패밀리 disambig 정답률을 크게 향상시키지만, 실행 시간이 문제. A패밀리만 선별 recovery(v38)로 70분 내 충족 가능함을 확인.

## 기술적 기여 요약

1. **소거법 프롬프트 공학**: 12개 규칙의 체계적 A/B 테스트 → 최적 조합
2. **Permutation Self-Consistency**: 선택지 순서 셔플 3회 + LLM arbiter
3. **반사실 디바이어싱**: 집단/성별 라벨 교환 → 편향 탐지 및 교정
4. **BBQ 구조 분석**: disambig 행동 주체 = 정답(99.7%) 패턴 발견
5. **시각 편향 분석**: 스톡사진 연출 포즈가 VLM 편향을 유발하는 메커니즘 규명
6. **규칙 준수 설계**: 모든 최종 답변이 LLM 생성임을 보장하는 아키텍처
7. **LLM 중재 디바이어싱**: base≠CF 충돌 시 3번째 LLM 패스로 공정 재판단
8. **Rule 11 양날의 검 발견**: disambig 도움/ambig 해로움 — 프롬프트 규칙의 의도치 않은 부작용 규명
9. **단계별 시각편향 경고**: base가 아닌 중재 단계에서만 B패밀리에 시각편향 경고 → ambig 보호 + B 과잉commit 억제 동시 달성
10. **Recovery 2단계 (binary choice)**: 1차 실패 건에 "ONLY two answers" 프롬프트로 2차 회수 → 99.8% recovery
11. **외부 논문 벤치마킹**: DEBIASLENS 등 최신 기법 대비 프롬프트 기반 접근의 상한 확인
